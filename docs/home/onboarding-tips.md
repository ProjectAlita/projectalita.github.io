## Category 3: Creating Your First Agent - Complete Workflow

### Tip 12: What Are Agents?

Agents are AI assistants with specific instructions and capabilities. They can search the web, create tickets, analyze data, or handle any task you define. Each agent combines an AI model with custom instructions and optional toolkits to become a specialized teammate!

**Quick Action:** In Chat, type # and browse existing agents to see examples.

![Agent examples in the assistant picker](../img/onboarding/tip-12-what-are-agents.png)

---

### Tip 13: Start Creating an Agent

In the Participants panel, click the **+** icon next to "Agents", then click "**+ Create new**". The Agent Canvas interface opensâ€”this is your visual editor for configuring everything about your agent!

**Quick Action:** Click + next to Agents > Click "+ Create new".

![Create new agent button in Participants panel](../img/onboarding/tip-13-create-agent-start.png)

---

### Tip 14: Fill Required Agent Fields

In the Canvas interface, start with the GENERAL section. Enter a **Name*** (e.g., "Code Review Assistant") and **Description*** (what your agent does). Required fields are marked with * and must be filled before you can save!

**Quick Action:** Fill in Name and Description fields in the GENERAL section.

![Agent creation form with Name and Description fields](../img/onboarding/tip-14-agent-required-fields.png)

---

### Tip 15: Write Agent Instructions

In the INSTRUCTIONS section, tell your agent exactly how to behave. Example: "You are a code reviewer that checks for security issues, performance concerns, and best practices. Always be constructive and specific." Clear instructions = better results!

**Quick Action:** Write detailed instructions in the INSTRUCTIONS section.

![Agent instructions field with example text](../img/onboarding/tip-15-agent-instructions.png)

---

### Tip 16: Add Welcome Message (Optional)

In the WELCOME MESSAGE section, write what users see when they first interact with your agent. Example: "Hello! I'm your code review assistant. Share your code and I'll help identify improvements!"

**Quick Action:** Add a friendly welcome message for your agent.

![Welcome message configuration in agent canvas](../img/onboarding/tip-16-agent-welcome.png)

---

### Tip 17: Add Conversation Starters

In the CONVERSATION STARTERS section, click "+ Starter" to add helpful prompts (maximum 4). Examples: "Review this code", "Check for security issues", "Explain best practices". Guide users on how to use your agent!

**Quick Action:** Click "+ Starter" and add 2-3 useful prompts.

![Conversation starters configuration showing add button](../img/onboarding/tip-17-agent-starters.png)

---

### Tip 18: Save Initial Configuration

After filling required fields and instructions, click the **Save** button. This creates your agent and opens the advanced configuration interface where you can select AI models, add toolkits, and fine-tune settings!

**Quick Action:** Click Save to proceed to advanced configuration.

![Save button in agent canvas](../img/onboarding/tip-18-agent-first-save.png)

In the advanced configuration, click "**Select LLM Model**" to choose which AI model powers your agent (GPT-4, Claude, etc.). Each model has different strengthsâ€”choose based on your agent's task!

## Category 4: Creating Pipelines - Multi-Step Workflows

### Tip 24: What Are Pipelines?

Pipelines are automated workflows that chain together multiple AI steps, decisions, loops, and tool calls. Perfect for complex processes like "read ticket â†’ analyze â†’ create subtasks â†’ update status"â€”all happening automatically!

**Quick Action:** In Chat, type # and browse existing pipelines to see examples.

![Pipeline examples in the assistant picker](../img/onboarding/tip-24-what-are-pipelines.png)

---

### Tip 25: Start Creating a Pipeline

In the Participants panel, click the **+** icon next to "Pipelines", then click "**+ Create new**". The Pipeline Canvas interface opens with Configuration and Flow Editor tabsâ€”this is where you build your automated workflows!

**Quick Action:** Click + next to Pipelines > Click "+ Create new".

![Create new pipeline button in Participants panel](../img/onboarding/tip-25-create-pipeline-start.png)

---

### Tip 26: Configure Pipeline Basics

In the Configuration tab, fill the GENERAL section with **Name*** (e.g., "Customer Feedback Pipeline") and **Description***. Add optional Welcome Message and Conversation Starters just like with agents. Configure step limit in ADVANCED section (default: 25 steps).

**Quick Action:** Fill Name and Description in GENERAL section.

![Pipeline configuration showing required fields](../img/onboarding/tip-26-pipeline-config.png)

---

### Tip 27: Save to Unlock Flow Editor

Click **Save** to create your pipeline. Important: The Flow Editor tab is disabled until you save! After saving, the advanced configuration opens where you can design your visual workflow in the Flow Editor tab.

**Quick Action:** Click Save to unlock the Flow Editor tab.

![Save button with Flow Editor tab disabled](../img/onboarding/tip-27-pipeline-save-first.png)

---

### Tip 28: Access the Flow Editor

After saving, click the **Flow Editor** tab to design your workflow visually. The Flow Editor uses a drag-and-drop canvas where you connect nodes to create your automation logic. This is where pipelines come to life!

**Quick Action:** Click the "Flow Editor" tab after initial save.

![Flow Editor tab in pipeline canvas](../img/onboarding/tip-28-flow-editor-tab.png)

---

### Tip 29: Add Nodes to Your Workflow

In Flow Editor, click **+ Add Node** to see available node types: Agent (AI conversations), Function (tool calls), Condition (if/else logic), Loop (repetition), Router (multiple paths), and more. Each node performs a specific workflow step!

**Quick Action:** Click "+ Add Node" to explore node types.

![Node palette showing different pipeline node types](../img/onboarding/tip-29-pipeline-nodes.png)

---

### Tip 30: Design Your Workflow Visually

Drag nodes onto the canvas and connect them by drawing lines between connection points. Each connection shows the flow of data and control through your pipeline. Build complex logic without writing code!

**Quick Action:** Drag an Agent node onto the canvas and connect it.

![Flow Editor with connected nodes showing workflow](../img/onboarding/tip-30-design-workflow.png)

---

### Tip 31: Configure Pipeline Model & Toolkits

Return to the Configuration tab to select the LLM model for your pipeline and add toolkits, nested agents, or nested pipelines. Pipelines can use everything agents canâ€”plus they orchestrate multiple steps!

**Quick Action:** Click Configuration tab, select model, add toolkits.

![Pipeline configuration tab with model and toolkit options](../img/onboarding/tip-31-pipeline-model-toolkits.png)

---

### Tip 32: Save and Test Your Pipeline

After designing your workflow, click **Save** and close the Canvas (X button). Your pipeline appears in the Participants panel. Add it to Chat with **#PipelineName**, activate it, and send a message to watch it execute each step!

**Quick Action:** Save pipeline, add to Chat with #, test it.

![Pipeline execution showing step-by-step progress in chat](../img/onboarding/tip-32-test-pipeline.png)

## Category 5: Toolkits & Credentials - Connect External Services

### Tip 33: What Are Toolkits?

Toolkits connect your agents and pipelines to external services like GitHub, Jira, Confluence, Slack, Azure DevOps, and more. Each toolkit provides pre-configured tools that your AI can use to read data, create items, or automate tasks in those services!

**Quick Action:** Navigate to Toolkits menu in sidebar to see available types.

![Toolkits menu showing available integration types](../img/onboarding/tip-33-what-are-toolkits.png)

---

### Tip 34: Create Credentials First

Before creating toolkits, you need credentials (API keys, tokens, passwords) to connect to external services. Go to Settings > Credentials and click "+ Create". Enter your API key/token, name it, and save. These credentials will be used by toolkits!

**Quick Action:** Go to Settings > Credentials > Click "+ Create".

![Credentials creation interface](../img/onboarding/tip-34-create-credentials.png)

---

### Tip 35: Start Creating a Toolkit

In the Participants panel, click **+** next to "Toolkits", then click "**+ Create new Toolkit**". The Toolkit Canvas opens where you'll select the service type and configure the connection!

**Quick Action:** Click + next to Toolkits > Click "+ Create new Toolkit".

![Create new toolkit button in Participants panel](../img/onboarding/tip-35-create-toolkit-start.png)

---

### Tip 36: Select Toolkit Type

In the toolkit creation interface, choose your service type from the dropdown (GitHub, Jira, Confluence, Slack, etc.). Each type comes pre-configured with relevant tools for that service. Fill in Name* and Description* (optional) of your toolkit.

**Quick Action:** Select toolkit type and fill required fields.

![Toolkit type selection showing available integrations](../img/onboarding/tip-36-toolkit-types.png)

---

### Tip 37: Configure Toolkit Connection

In the CONFIGURATION section, select the credential you created earlier from the "Credentials" dropdown. Add any additional settings like API endpoints, project keys, or database connections specific to that toolkit type.

**Quick Action:** Select credential from dropdown, configure settings.

![Toolkit configuration showing credential selection](../img/onboarding/tip-37-toolkit-configuration.png)

---

### Tip 38: Enable Specific Tools

After configuration, scroll to the TOOLS section. Check only the specific tools your agent will needâ€”this improves security and performance. For example, in a GitHub toolkit, enable "create_issue" but maybe not "delete_repository"!

**Quick Action:** Review and select only needed tools in TOOLS section.

![Tools selection showing checkboxes for individual tools](../img/onboarding/tip-38-enable-tools.png)

---

### Tip 39: Save Your Toolkit

Click **Save** to create your toolkit. It immediately appears in the Participants panel under "Toolkits" and is ready to be added to agents, pipelines, or used directly in conversations!

**Quick Action:** Click Save, then close Canvas with X button.

![Newly created toolkit in Participants panel](../img/onboarding/tip-39-toolkit-created.png)

---

### Tip 40: Add Toolkit to an Agent

To give an agent access to a toolkit, hover over the agent in Participants and click the **Edit** icon. In the agent Canvas, scroll to the TOOLKITS section, click "+ Toolkit", select your toolkit, and save. Your agent now has those capabilities!

**Quick Action:** Edit agent > TOOLKITS section > + Toolkit > Select > Save.

![Adding toolkit to agent in TOOLKITS section](../img/onboarding/tip-40-add-toolkit-to-agent.png)

---

### Tip 41: Toolkits Work Automatically in Chat

When you add a toolkit to the Participants panel (click + next to Toolkits), it stays there and the AI automatically uses its tools when needed based on your conversation. No need to activate itâ€”just ask, and the AI selects the right tools!

**Quick Action:** Add a toolkit to Participants, then ask questions that need it.

![Toolkit in Participants panel being used automatically](../img/onboarding/tip-41-toolkit-in-chat.png)
